Component-Level Signals

What is a Signal?

Signals are Angular’s built-in way to manage reactive state — think of them as reactive variables. When a signal’s value changes, Angular automatically updates everything that depends on it.

  • Lightweight and simple
  • Fully tracked by Angular
  • Works smoothly with OnPush change detection

You can:

  • Create a signal: const count = signal(0)
  • Read the value by calling it: count()
  • Update it: count.set(1) or count.update(v => v + 1)

Unlike a regular let, Signals notify Angular when they change — so your UI stays in sync automatically.
