Dynamic Settings Page – System Design Task

Build a dynamic and role-aware settings page that fetches a list of settings from a mock backend. The settings should be organized visually, efficiently rendered, and restricted by user roles.

Task

  • Each setting has: type, key, label, value, group, and roles: string[].
  • Render each setting under its group title, using a collapsible panel per group.
  • Each setting is editable by the user using its respective input type (e.g. toggle, dropdown, text input), and updates should be tracked locally.
  • Only display settings visible to the current user role (e.g. 'admin', 'editor').
  • Each setting type (e.g. toggle, select, input) should be rendered dynamically.
  • Show a visual indicator (e.g. icon or color) if a setting was modified (dirty).
  • Optimize your data structure for performance (e.g. fast access by group, key, role).
  • Do not hardcode types or group names — the system must be extensible.
  • Use a “Save” button to log changed settings to the console (no backend needed).
